# ADR-0001 — Retire `SKILL.digest.md` and `SKILL.minimal.md` tier system

> Status: **Accepted** — landed in branch `feat/skills-sensei` (May 2026).
> Companion: [findings.md](./findings.md), [implementation-plan.md](./implementation-plan.md).

## Context

Skills in this repo previously shipped in three tiers per directory:

- `SKILL.md` — full body, reserved for authoring/debugging.
- `SKILL.digest.md` — generator-produced compression, default for runtime.
- `SKILL.minimal.md` — generator-produced compression, escalation at >80%
  context utilization.

The three tiers were auto-generated by
[`tools/scripts/generate-skill-digests.mjs`](../../../tools/scripts/generate-skill-digests.mjs)
and validated by `validate-skill-checks.mjs`. Every agent body, prompt, and
several instruction files hard-coded reads against the digest tier; the minimal
tier was the documented escalation path.

A May 2026 deep audit (see [findings.md](./findings.md)) surfaced systemic
problems with the tier system:

1. **The minimal tier was structurally broken.** The generator's
   `generateMinimal` heuristic emitted only an H2 label and the first
   non-table/list/blockquote line per section (up to 120 chars). For skills
   whose bodies started with tables, the output was section labels with no
   body. Examples:
   - `azure-rbac/SKILL.minimal.md` was 3 lines total.
   - `azure-deploy/SKILL.minimal.md` was 24 lines, half-empty section
     placeholders.
   - `drawio/SKILL.minimal.md` had leading lines truncated mid-clause.
2. **Digests routinely truncated mid-table or mid-bullet.** The
   `trimSection` heuristic cut at a per-section line cap and appended
   `> _See SKILL.md for full content._`, severing semantic units.
3. **Drift compounded silently.** The generator's default behaviour was
   to skip skills whose digest already existed, so source edits never
   regenerated the digest until someone manually invoked `generate:digests`.
4. **Three sources to maintain.** Authors had to keep three files in sync
   per skill plus the generator heuristics. A 33-skill catalog meant 99
   files plus tooling. In practice, only `SKILL.md` was hand-edited; the
   tiers drifted.

## Decision

Retire both `SKILL.digest.md` and `SKILL.minimal.md`. Every skill is now
**a single file**: `.github/skills/<name>/SKILL.md`. Deep content stays in
`references/*.md` and is loaded on demand.

## Consequences

### Positive

- **Single source of truth per skill.** Authors edit one file; readers
  load one file. No regeneration step.
- **Net −3.4K lines removed.** 66 generated files deleted; +738 / −4144
  lines on the PR (most deletions are tier content, most additions are
  one-line wiring updates).
- **Tooling surface shrinks.** Deleted: `generate-skill-digests.mjs` (~200
  lines) and the digest portion of `validate-skill-checks.mjs` (~60 lines).
  Trimmed: `safe-shell.mjs` `SKILL_FILE_NAMES`, the orphan-content regex,
  the orphan-skill-discovery test fixtures, `package.json` `generate:digests`,
  the `applyTo` glob in `no-interactive-shell.instructions.md`. Net:
  zero new validators, two old ones simplified.
- **Discoverability unchanged.** Copilot's skill auto-discovery still reads
  only frontmatter `name` + `description`; the body is loaded on demand.
  The skill's runtime cost is unchanged because agents never loaded all
  three tiers anyway.
- **Drift gone.** With one file per skill, the tier-mismatch validator can
  retire. The only remaining drift surface is `SKILL.md` ↔ `references/*.md`,
  which is the same as it was before tiers existed.

### Negative

- **Larger startup loads for agents that read full skills.** Agents that
  previously loaded `SKILL.digest.md` (~150-320 tokens) now load `SKILL.md`
  (400-800 tokens) on first load. Mitigations:
  - Agents read each `SKILL.md` once per session (already the rule).
  - Predecessor-artifact reads — the much larger context cost — are still
    governed by the `context-management` skill's Mode A artifact tiers
    (`full` / `summarized` / `minimal`), which now apply to
    `agent-output/` artifacts only, not to skills.
  - Skills that exceeded ~500 lines were already split via `references/`
    per `agent-skills.instructions.md`. The post-retirement median skill
    body is ~120 lines; the largest is 196 (`azure-defaults`).
- **High-utilization fallback is now content-aware compaction, not a
  filesystem switch.** Previously the `>80% utilization` rule said "switch
  to `SKILL.minimal.md`"; that was a no-op because the minimal tier
  carried no real content. The replacement rule is: "Stop loading
  additional skills; rely on summaries already in context; use
  `apex-recall show <project> --json` for cached lookups." This is what
  agents were already doing in practice; the rule now matches reality.

### Neutral

- **Site changelog history retained.** Past entries mentioning
  `SKILL.digest.md` / `SKILL.minimal.md` are historical records and stay
  intact per [docs-trigger.instructions.md](../../../.github/instructions/docs-trigger.instructions.md).
- **Validation surface unchanged externally.** `npm run validate:all`
  still runs the same job names; only the digest checks inside
  `validate-skill-checks.mjs` are gone.

## Alternatives considered

### A — Hand-author all `SKILL.minimal.md` files

Highest content quality. Authoring cost grows linearly with skill count
(33 skills × ~25 lines minimal = ~800 hand-authored lines). Maintenance
cost is permanent: every `SKILL.md` edit forces a `SKILL.minimal.md`
review.

**Rejected**: the authoring + maintenance cost outweighs the marginal
context-window saving, especially because the audit showed agents already
re-read skills on doubt instead of trusting compressed variants.

### B — Rebuild the generator to be content-aware

Detect tables, code fences, lists; prefer narrative paragraphs; refuse
to emit a section unless it has a real body. Medium authoring cost
(generator becomes ~400 lines), zero per-skill maintenance.

**Rejected**: even a content-aware generator does not solve the
fundamental problem that *agents prefer the canonical source when in
doubt*. The minimal-tier load pattern was aspirational, not real. A
better generator would produce nicer-looking files that agents would
still bypass.

### C — Retire the tier system (chosen)

Lowest authoring + maintenance cost. Trades a small worst-case
context-load increase for unconditional source-of-truth clarity.

**Accepted.** See "Consequences" above.

## Implementation

Two-commit PR on branch `feat/skills-sensei`:

- **Commit 1**: delete 66 tier files, delete generator, trim validator
  + safe-shell + orphan-content regex + orphan-skill-discovery test,
  remove `generate:digests` from `package.json`.
- **Commit 2**: rewrite ~30 agent / prompt / instruction / site-doc
  references from `SKILL.digest.md` → `SKILL.md`; rewrite tier-discussion
  sentences in agents and `context-optimization.instructions.md`; rewrite
  the `Skill Loading` section in `context-management/SKILL.md` to point
  the artifact tier system only at `agent-output/`; add canonical
  `## Azure Defaults (canonical)` to `.github/copilot-instructions.md`
  and link `azure-defaults/SKILL.md` back to it; rewrite `azure-rbac`
  and `azure-storage` on `az CLI` + `microsoft-learn` MCP; add `INVOKES:`
  clauses across 9 skills; add `validate:region-canonical` drift validator.

## Verification

After the change:

- `find .github/skills -name "SKILL.digest.md" -o -name "SKILL.minimal.md"`
  → 0 results.
- `grep -rn "SKILL\.digest\|SKILL\.minimal" .github AGENTS.md README.md
  package.json lefthook.yml tools/ site/src site/public`
  → only the legacy-tier rejection comment in
  `validate-orphaned-content.mjs` and the changelog history entries.
- `npm run validate:all` → 0 errors, 0 warnings.
