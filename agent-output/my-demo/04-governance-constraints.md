## 🚫 Deployment Blockers

> [!CAUTION]
> **2 blockers identified — both have resolutions planned.**

### MFA Enforcement for Resource Write Actions

- **Policy ID**: `sys.mfa-write`
- **Effect**: Deny
- **Scope**: Management Group
- **Enforcement Mode**: Default
- **Impact**: Blocks ARM write operations from non-MFA user identities
- **Assessment Date**: 2026-02-24

**Resolution**: Deploy via service principal or ensure deploying user has MFA enabled.
The deployment script (`deploy.ps1`) will support both interactive (MFA) and SP authentication.

**Status**: ✅ **RESOLVED** — deployment script supports service principal auth

### JV-Enforce Resource Group Tags v3

- **Policy ID**: `jv-enforce-rg-tags-v3`
- **Effect**: Deny
- **Scope**: Management Group
- **Enforcement Mode**: Default
- **Impact**: Resource group creation/update fails without all 9 required tags
- **Assessment Date**: 2026-02-24

**Resolution**: Include all 9 required tags as parameters in `main.bicep`. Default values provided for demo context.

**Status**: ✅ **RESOLVED** — all 9 tags included in implementation plan

## 🏷️ Required Tags

> [!WARNING]
> Azure Policy requires **9 tags** on resource groups (not just the 4 baseline tags).
> Tag inheritance policy auto-propagates these to child resources.

All resources must include the following tags:

```bicep
tags: {
  environment: environment         // dev, staging, prod
  owner: owner                     // Team or individual
  costcenter: costCenter           // Cost center code
  application: 'my-demo'          // Application name
  workload: 'demo'                // Workload type
  sla: 'best-effort'             // SLA tier
  'backup-policy': 'none'        // Backup policy
  'maint-window': 'anytime'      // Maintenance window
  'technical-contact': owner      // Technical contact
}
```
