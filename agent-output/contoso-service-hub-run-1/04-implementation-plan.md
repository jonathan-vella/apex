## 📋 Overview

This implementation plan defines the Bicep Infrastructure-as-Code structure for the **Contoso Service Hub** — an enterprise digital platform deployed to **swedencentral** across 3 environments (dev, staging, production). The plan uses **Azure Verified Modules (AVM)** for all resources where available, follows **CAF naming conventions**, and enforces governance constraints discovered from the live Azure tenant.

| Parameter                | Value                                          |

| **IaC Tool**             | Bicep (AVM-first)                              |
| **Region**               | swedencentral                                  |
| **Environments**         | dev, staging, prod                             |
| **Resource Count**       | 16 Azure resources + supporting infrastructure |
| **Architecture Pattern** | N-Tier Web + AKS Container (Enterprise)        |

### Governance Summary

| Category                       | Count | Impact                                                                                     |
| ------------------------------ | ----- | ------------------------------------------------------------------------------------------ |
| Deny policies (blockers)       | 1     | MFA enforcement on write operations — automation identity must be MFA-exempt               |

## 🔧 Parameter Design

### Environment-Specific Parameters

The `main.bicepparam` file uses a single `environment` parameter to drive all sizing decisions via a lookup map in `main.bicep`. This eliminates parameter file duplication.

```yaml
# main.bicepparam structure
using './main.bicep'

param environment = '{env}'        # Replaced per deployment: dev | staging | prod
param location = 'swedencentral'
param projectName = 'contoso-service-hub'
param deployPhase = 0              # 0 = all phases
param ownerTag = 'Contoso Platform Team'
param costCenterTag = 'CC-4200-Digital'
```
