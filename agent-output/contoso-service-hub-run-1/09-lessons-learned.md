## Key Findings

### 1. Agent Schema Compliance (LL-001)

**Step 1 — Session state decision_log format mismatch**

The Requirements agent generated decision_log entries with a non-standard ID pattern (`DL-xxx` instead of `Dxxx`) and missing required fields (`agent`, `title`, `choice`). This was caught by the E2E step validator and self-corrected in 1 iteration.

**Recommendation**: Include exact decision_log JSON schema example in subagent prompts to prevent format drift.

### 2. Deprecated Service Selection (LL-002, LL-005)

**Step 1 — Azure AD B2C for greenfield 2026 project**

The Requirements agent defaulted to Azure AD B2C despite Microsoft discontinuing it for new customers in May 2025. The correct choice for this greenfield project is Microsoft Entra External ID.

**Step 2 — Redis Enterprise E50 retires within contract term**

The Architecture agent selected Redis Enterprise E50 with a 3-year RI, but Enterprise E50 retires March 31, 2027 — within the 3-year contract window (2026-2029). Replaced with Azure Managed Redis M100.

**Recommendation**: Add automatic deprecation/retirement date checking in Architecture agent for any multi-year commitment. Maintain a deprecated-services list in the azure-defaults skill.

### 3. Reliability Design vs. Cost Optimization (LL-006, LL-007)

**Step 2 — APIM Standard v2 breaks zone-redundant critical path**

The Architecture agent selected the cheaper Standard v2 tier to optimize costs, but this tier does not support availability zones. Since APIM sits in the critical request path (Front Door → APIM → AKS), zone-redundant APIM Premium v2 is required for the stated 99.9% composite SLA.

**Step 2 — AKS 2-node baseline cannot cover 3 availability zones**

Architecture specified 3-AZ spread but only provisioned 2 user nodes, leaving one AZ without user-node capacity at baseline. Fixed to 3 minimum user nodes (1 per AZ).

**Recommendation**: Add automated validation: if a service claims zone-redundancy in the WAF assessment, verify the selected SKU supports AZ and the baseline node count ≥ AZ count.

### 4. Documentation Completeness (LL-008, LL-009, LL-010)

**Step 7 — APIM missing from resource inventory**

The As-Built agent generated a resource inventory that omitted APIM despite it being documented in all prior steps. It was categorized as an edge concern rather than given a separate row.

**Step 7 — GDPR compliance matrix lacked clause-by-clause RFQ Section 4.3 mapping**

The original compliance matrix used broad GDPR statements. The challenger identified that RFQ Section 4.3 has specific clauses for logs, backups, metadata, telemetry, caching, indexing, and remote support that each need explicit control mapping.

**Step 7 — DR plan omitted APIM and identity configuration recovery**

The backup/DR plan documented RTO targets for APIM but had no export/import procedure for APIs, policies, products, or Entra External ID configuration state.

**Recommendation**: As-Built agent should cross-reference the AVM matrix and implementation plan to ensure 1:1 resource coverage in inventory. DR plans should cover both infrastructure (IaC-recoverable) and configuration state (manual export/import).
