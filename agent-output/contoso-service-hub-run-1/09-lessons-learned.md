## Critical Lessons

### LL-003: Azure Front Door EU Data Boundary Exclusion (Step 2)

**Severity**: Critical

The Architecture agent recommended Azure Front Door for CDN/edge without
noting its exclusion from the EU Data Boundary. The RFQ Section 4.3
explicitly prohibits processing, caching, or indexing outside the EU
without written approval. Front Door uses global anycast routing and
edge caching that may process data outside EU.

**Resolution**: Challenger review caught this. Architecture was corrected
to recommend Application Gateway (regional, fully EU-contained) as the
compliant baseline, with Front Door as an option requiring explicit
Contoso exception.

**Recommendation**: Add EU Data Boundary compatibility check to the
Architect agent for GDPR/EU-residency projects. Maintain a reference
list of Azure services excluded from EU Data Boundary.

## High Severity Lessons

### LL-001: Decision Log Format Mismatch (Step 1)

The Requirements agent generated `decision_log` entries with incorrect
ID format (`DL-001` vs required `D001`) and missing required fields
(`agent`, `title`, `choice`). Fixed after session state validation
failure. **Add schema examples to agent prompts.**

### LL-002: Invented NFRs Not in Source RFP (Step 1)

Requirements agent introduced concrete targets (RTO 4h, RPO 1h,
page load under 2s) not stated in the RFQ. These were presented as
extracted requirements rather than assumptions. **Mark non-source
values as assumptions.**

### LL-006: Entra External ID MFA Sovereignty (Step 2)

Architecture assessment approved Entra External ID with conditional
MFA without noting that PSTN/SMS MFA can be processed outside the EU.
Challenger flagged this. **Constrain MFA to FIDO2/TOTP for EU Data
Boundary compliance.**

## Medium Severity Lessons

### LL-004: AVM Parameter Schema Mismatches (Step 5)

PostgreSQL, Redis, and Storage modules required 5 build iterations
because AVM parameter APIs differ from raw ARM APIs. The agent used
ARM-style parameters instead of AVM-specific abstractions. **Pre-compute
AVM parameter schemas in the AVM matrix.**

### LL-005: App Gateway AVM Incompatibility (Step 5)

AVM Application Gateway module 0.9.0 uses an abstracted API incompatible
with full WAF_v2 configuration, requiring fallback to native ARM resource
declaration. **Flag AVM modules with known limitations in the AVM matrix.**
