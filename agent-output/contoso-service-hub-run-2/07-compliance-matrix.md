## 🗺️ 1. Control Mapping

### GDPR Controls

| # | Control | Requirement | Implementation | Status |
| --- | --- | --- | --- | --- |
| G-01 | Data Residency | All data must remain in EU | All resources in swedencentral; ZRS (no GRS); App Gateway replaces Front Door | ✅ Implemented |
| G-02 | Encryption at Rest | Personal data encrypted at rest | Azure-managed encryption (AES-256) on all storage, PostgreSQL, Redis | ✅ Implemented |
| G-03 | Encryption in Transit | TLS for all data movement | TLS 1.2 minimum on all services (POL-002, POL-005, POL-012, POL-014) | ✅ Implemented |
| G-04 | Access Control | Least-privilege access | RBAC via Managed Identity; no shared keys; Entra ID authentication | ✅ Implemented |
| G-05 | Data Minimization | Collect only necessary data | Application-layer concern — infrastructure supports via separate storage containers | ⚠️ App-layer |
| G-06 | Right to Erasure | Delete personal data on request | PostgreSQL supports row deletion; Blob versioning with soft delete | ⚠️ App-layer |
| G-07 | Breach Notification | 72-hour notification requirement | Azure Monitor alerts + Log Analytics; App Insights anomaly detection | ✅ Monitoring in place |
| G-08 | Data Processing Records | Maintain processing activity records | Log Analytics workspace with 365-day retention (prod) | ✅ Implemented |
| G-09 | DPO Support | Support Data Protection Officer | Audit logs accessible via Log Analytics; Key Vault access logs | ✅ Implemented |
| G-10 | Network Isolation | Protect personal data networks | Private Endpoints for all data services; NSG deny-all defaults | ✅ Implemented |
| G-11 | Key Management | Secure key storage | Key Vault with RBAC, purge protection, private endpoint | ✅ Implemented |
| G-12 | Audit Logging | Comprehensive audit trail | Diagnostic settings on all resources → Log Analytics | ✅ Implemented |

### PCI-DSS Controls (Applicable to Payment Processing)

| # | Control | Requirement | Implementation | Status |
| --- | --- | --- | --- | --- |
| P-01 | Network Segmentation | Isolate cardholder data environment | Dedicated subnets with NSGs; Private Endpoints for data services | ✅ Implemented |
| P-02 | Firewall / WAF | Protect public-facing apps | App Gateway WAF v2, Prevention mode, OWASP CRS 3.2 | ✅ Implemented |
| P-03 | Encryption | Encrypt cardholder data | TLS 1.2+ in transit; AES-256 at rest | ✅ Implemented |
| P-04 | Access Control | Restrict access to cardholder data | Managed Identity; no shared credentials; RBAC | ✅ Implemented |
| P-05 | Monitoring | Track all access to cardholder data | Log Analytics + App Insights with 365-day retention | ✅ Implemented |
| P-06 | Vulnerability Scanning | Regular security assessments | Microsoft Defender for Cloud (to be enabled post-deployment) | ⚠️ Post-deploy |
| P-07 | Secure Development | Secure SDLC practices | GitHub Advanced Security (SAST/DAST) — configured in CI/CD | ⚠️ CI/CD config |
| P-08 | Key Rotation | Regular key and certificate rotation | Key Vault supports rotation; automation to be configured | ⚠️ Post-deploy |
| P-09 | Tokenization | Tokenize card numbers | Application-layer concern (payment gateway integration) | ⚠️ App-layer |
| P-10 | MFA | Multi-factor authentication for admin | Entra ID MFA (FIDO2/TOTP per ADR-003) + Bastion | ✅ Implemented |

| P-12 | Penetration Testing | Annual penetration testing | Planned — not infrastructure-scoped | ⚠️ Planned |
