## 🚨 2. Incident Response

### Severity Levels

| Level | Definition | Response Time | Escalation |
| --- | --- | --- | --- |
| SEV-1 | Complete service outage | 15 minutes | Immediate — all hands |
| SEV-2 | Degraded performance (>50% users affected) | 30 minutes | Platform team lead |
| SEV-3 | Partial feature outage | 2 hours | On-call engineer |
| SEV-4 | Minor issue, no user impact | Next business day | Backlog triage |

### Common Incident Playbooks

#### AKS Cluster Unresponsive

1. Check cluster status: `az aks show -g rg-contoso-hub-prod -n aks-contoso-hub-prod --query powerState`
2. Check node health: `kubectl get nodes`
3. Check system pods: `kubectl get pods -n kube-system`
4. Review cluster logs: Azure Portal → AKS → Diagnose and solve problems
5. If nodes are `NotReady`: `az aks nodepool scale -g rg-contoso-hub-prod --cluster-name aks-contoso-hub-prod -n systempool --node-count 3`
6. Escalate if unresolved after 30 minutes

#### PostgreSQL Connection Failures

1. Verify server status: `az postgres flexible-server show -g rg-contoso-hub-prod -n psql-contoso-hub-prod --query state`
2. Check private endpoint connectivity from AKS pod: `kubectl exec -it test-pod -- nslookup psql-contoso-hub-prod.postgres.database.azure.com`
3. Review connection limits: `az postgres flexible-server show -g rg-contoso-hub-prod -n psql-contoso-hub-prod --query "storage.storageSizeGb"`
4. Check for blocking queries via Azure Portal → PostgreSQL → Query Performance Insight
5. If HA failover needed: `az postgres flexible-server restart -g rg-contoso-hub-prod -n psql-contoso-hub-prod --failover Forced`

#### App Gateway 502 Errors

1. Check backend health: `az network application-gateway show-backend-health -g rg-contoso-hub-prod -n agw-contoso-hub-prod`
2. Verify AKS pods are running: `kubectl get pods -n production`
3. Check health probe configuration in Azure Portal → App Gateway → Health probes
4. Review WAF logs: Azure Portal → App Gateway → Diagnostic settings → WAF logs
5. If WAF is blocking legitimate traffic: Review WAF rules, add exclusions if needed

## 🔧 3. Common Procedures

### Scale AKS Node Pool

```bash
# Scale system pool to 5 nodes
az aks nodepool update \
  -g rg-contoso-hub-prod \
  --cluster-name aks-contoso-hub-prod \
  -n systempool \
  --enable-cluster-autoscaler \
  --min-count 3 \
  --max-count 10
```

### Rotate Key Vault Secrets

```bash
# List current secrets
az keyvault secret list --vault-name kv-contoso-prod-{suffix} --query "[].name"

# Set new secret version
az keyvault secret set --vault-name kv-contoso-prod-{suffix} --name {secret-name} --value "{new-value}"

# Restart pods to pick up new secret (if using CSI driver)
kubectl rollout restart deployment/{app-name} -n production
```

### PostgreSQL Maintenance

```bash
# Check storage utilization
az postgres flexible-server show -g rg-contoso-hub-prod -n psql-contoso-hub-prod \
  --query "{StorageGB: storage.storageSizeGb, Tier: sku.tier, Compute: sku.name}"

# Scale up compute (requires restart)
az postgres flexible-server update -g rg-contoso-hub-prod -n psql-contoso-hub-prod \
  --sku-name Standard_D8ds_v5
```

### Update Bicep Infrastructure

```bash
# 1. Make changes to Bicep templates
# 2. Validate
bicep build infra/bicep/contoso-service-hub-run-2/main.bicep

# 3. Preview changes
az deployment group what-if \
  -g rg-contoso-hub-prod \
  -f infra/bicep/contoso-service-hub-run-2/main.bicep \
  -p infra/bicep/contoso-service-hub-run-2/main.bicepparam

# 4. Deploy (after approval)
az deployment group create \
  -g rg-contoso-hub-prod \
  -f infra/bicep/contoso-service-hub-run-2/main.bicep \
  -p infra/bicep/contoso-service-hub-run-2/main.bicepparam
```
