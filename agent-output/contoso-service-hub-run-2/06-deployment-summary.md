### Step 0 — Create resource group with mandatory governance tags

> The live Deny policies (LIVE-002 and LIVE-003) block resource-group creation when any of the
> **9 mandatory tag keys** is absent. The `az group create` command below includes all required tags.

```bash
az group create \
  --name rg-contoso-hub-dev \
  --location swedencentral \
  --tags \
    environment=dev \
    owner=Contoso \
    costcenter=CC-PLATFORM \
    application=contoso-service-hub \
    workload=platform-engineering \
    sla=tier-2 \
    "backup-policy=daily-7d" \
    "maint-window=sat-02:00-04:00" \
    "technical-contact=platform-ops@contoso.com" \
    "tech-contact=platform-ops@contoso.com"
```

<details>
<summary><strong>🚀 Option 2: Azure CLI — cumulative phased deployment</strong></summary>

> Phases are **cumulative**: selecting a later phase always reruns all previous phases idempotently.
> All parameters are sourced from `main.bicepparam`; `deployPhase` is overridden per run.
> Override `vmSshPublicKey` only when you need operator-controlled access to the management VM.
> Run from the workspace root.

```bash
# ── Phase 1: Foundation (monitoring, networking, identity, keyvault, budget) ──
az deployment group what-if \
  --resource-group rg-contoso-hub-dev \
  --template-file infra/bicep/contoso-service-hub-run-2/main.bicep \
  --parameters infra/bicep/contoso-service-hub-run-2/main.bicepparam \
               deployPhase=foundation \
  --validation-level Provider

# Review what-if output, then deploy:
az deployment group create \
  --resource-group rg-contoso-hub-dev \
  --template-file infra/bicep/contoso-service-hub-run-2/main.bicep \
  --parameters infra/bicep/contoso-service-hub-run-2/main.bicepparam \
               deployPhase=foundation \
  --name contoso-hub-foundation-$(date +%Y%m%dT%H%M%S) \
  --output table

# ── Phase 2: Data (cumulative — includes foundation) ──────────────────────────
az deployment group create \
  --resource-group rg-contoso-hub-dev \
  --template-file infra/bicep/contoso-service-hub-run-2/main.bicep \
  --parameters infra/bicep/contoso-service-hub-run-2/main.bicepparam \
               deployPhase=data \
  --name contoso-hub-data-$(date +%Y%m%dT%H%M%S) \
  --output table

# ── Phase 3: Edge (cumulative — includes foundation + data) ───────────────────
az deployment group create \
  --resource-group rg-contoso-hub-dev \
  --template-file infra/bicep/contoso-service-hub-run-2/main.bicep \
  --parameters infra/bicep/contoso-service-hub-run-2/main.bicepparam \
               deployPhase=edge \
  --name contoso-hub-edge-$(date +%Y%m%dT%H%M%S) \
  --output table

# ── Phase 4: Platform (cumulative — all phases) ───────────────────────────────
# Override vmSshPublicKey when you need an operator-controlled SSH key:
az deployment group create \
  --resource-group rg-contoso-hub-dev \
  --template-file infra/bicep/contoso-service-hub-run-2/main.bicep \

               deployPhase=platform \
               vmSshPublicKey='ssh-rsa AAAA... your-real-public-key contoso-mgmt-vm' \
```
